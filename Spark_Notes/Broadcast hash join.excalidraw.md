Broadcast Hash Join  ^uBb9bbc4

Broadcast  join is used when one table is small and other table is big 

How small:  memory less then 10MB is small table for default spark 

The logic behind this is that suppose we have a 1gb table and one 10Mb table

the 10Mb table will have only one partition since default one partition is 128mb

1gb table will have 1000/128 comes up to 8 partitions what we do is instead of 
shuffling we just send the 10mb table in every executor that is we broadcast it
by this we avoid shuffling. In original method if we had to shuffle we have to 
bring all our data in one executor only because table one has only 1 partition this
would choke the executor there what we do is we broadcast it that is send this 
10mb table into every executor where partitions of 1gb table is present by this
the joining will happen in executors and then we will get the clubbed result 

this is the optimal approach when one table is small and the other is big.
